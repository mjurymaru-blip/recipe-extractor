# AIレビューフィードバック (Phase 5 完了時点)

## 概要
- **レビュワー**: ChatGPT, Gemini
- **日時**: 2026-02-01
- **対象**: Phase 5 実装完了時点の全コード

## 総合評価
- **完成度**: 85〜90%
- **アーキテクチャ**: CLI（データ生成）とPWA（閲覧）の責務分離が高く評価された。
- **データ設計**: JSON中心設計は将来の拡張性（キャンプPWA連携など）に有利。
- **UX**: レシピ閲覧体験は良いが、動画プレイヤーの挙動に改善の余地あり。

## 改善提案

### 優先度：高（必須）
- **UX: 動画プレイヤーの永続化**
  - 現状: ステップ移動のたびにiframeが再生成・リロードされ、チラつきや通信が発生。
  - 対策: iframeを維持し、`postMessage` 等でシークのみ更新する実装に変更する。

### 優先度：中（推奨）
- **Refactor: カテゴリ定義の一元化**
  - 現状: `app.js`, `index.html`, `cli/index.js` に分散。
  - 対策: `constants.js` または共通設定ファイルを作成し、定義を一箇所にまとめる。
- **Refactor: app.jsの分割**
  - 現状: 1ファイルにロジックが集中しつつある。
  - 対策: `RecipeList`, `StepView`, `Filter` など機能ごとにモジュール分割。

### 優先度：低（将来）
- **Perf: JSONデータの分割読み込み**
  - 現状: 全件ロード方式。レシピ数増加時に遅くなる懸念。
  - 対策: `list.json` と詳細JSONに分割し、遅延ロードする。
- **Feature: 動画プレイヤーのUX改善**
  - 提案: デフォルトは小さく/折りたたみ、タップで強調など、手順を主役にするUI調整。

## アクションプラン
Phase 6（オフライン対応）の前に、**UX改善（動画プレイヤー永続化）** と **リファクタリング（カテゴリ一元化）** を実施することを推奨。
